Buffer Overflow چیست؟

تعریف:
Buffer Overflow زمانی اتفاق می‌افته که برنامه، داده‌ای بزرگ‌تر از ظرفیت اختصاص داده‌شده به یک متغیر (مثلاً آرایه) را در آن می‌ریزد.

چرا خطرناکه؟
چون اطلاعات اضافه، حافظه‌ی مجاور را خراب می‌کنه — مثلاً ممکنه:

متغیرهای دیگر را تغییر بده

مسیر اجرای برنامه را عوض کند (با تغییر Return Address)

کد دلخواه اجرا کند (مانند Shell)

 بخش ۲: ساختار حافظه در یک برنامه C
وقتی یک برنامه اجرا می‌شود، فضای حافظه به صورت تقریبی به شکل زیر تقسیم می‌شود:

sql
Copy code
+--------------------+ ← آدرس‌های بالا (بزرگ‌تر)
|    Stack           | ← اطلاعات تابع‌ها و متغیرهای محلی
+--------------------+
|    Heap            | ← حافظه داینامیک مثل malloc
+--------------------+
|    Data Segment    | ← متغیرهای global و static
+--------------------+
|    Text Segment    | ← کدهای اجرایی برنامه
+--------------------+ ← آدرس‌های پایین (کوچک‌تر)
🔸 Stack چیست؟
Stack برای ذخیره‌سازی:

پارامترهای تابع

متغیرهای محلی

آدرس بازگشت (Return Address)

Stack ساختار LIFO (Last-In-First-Out) داره.

🔸 Heap چیست؟
Heap برای اختصاص حافظه در زمان اجرا استفاده می‌شه، معمولاً با malloc, calloc, new و ...
