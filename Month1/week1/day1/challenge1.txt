سؤال: با توجه به آدرس‌هایی که گرفتی، آیا آدرس Heap از آدرس Global بالاتره یا پایین‌تره؟ چرا این موضوع مهمه توی حمله‌های BOF؟

مقایسه آدرس‌ها:
آدرس global: 0x57bfe99e6010

آدرس heap: 0x57c003f112a0

حالا اگر دقت کنیم:

Copy code
0x57c003f112a0 > 0x57bfe99e6010
یعنی heap بالاتر از global قرار گرفته — این دقیقاً برخلاف چیزی بود که گفتی (که گفتی less than).

🧠 نکته خیلی مهم:
در معماری حافظه لینوکس (و اکثر سیستم‌ها):

Code segment → پایین‌ترین آدرس‌ها

Data segment (global/static) → کمی بالاتر

Heap → رشد به سمت بالا (بالاتر از data)

Stack → در انتهای فضای آدرس و رشد به پایین

💡 چرا این مهمه توی BOF؟
چون اگر بخوای مثلاً shellcode رو داخل heap بریزی، و return address رو توی stack با آدرس heap بازنویسی کنی، باید بدونی آدرس دقیق heap کجاست. پس درک جایگاه بخش‌های مختلف حافظه خیلی مهمه برای:

Exploit نوشتن

ساختن payload

پرش به shellcode
