چالش نهایی روز اول:
📌 چالش 1:
در کد زیر، اگر کاربر بیش از 16 کاراکتر وارد کند چه اتفاقی می‌افتد؟ دقیق توضیح بده:

c
Copy code
#include <stdio.h>

int main() {
    char input[16];
    printf("Enter something:\n");
    gets(input);
    printf("You said: %s\n", input);
    return 0;
}
چرا ممکنه این کد crash کنه یا رفتار خطرناک داشته باشه؟

چه بخشی از حافظه ممکنه بازنویسی بشه؟




✅ پاسخ 

چون کاربر ممکنه بیش از ۱۶ کاراکتر وارد کنه، مقدار ورودی از اندازه‌ی آرایه‌ی input[16] بیشتر می‌شه. تابع gets() هم چون هیچ بررسی‌ای روی اندازه انجام نمی‌ده، داده‌های اضافی رو توی حافظه‌ی بعدی می‌نویسه و این باعث Buffer Overflow می‌شه.

نتیجه:

ممکنه متغیرهای محلی دیگه، آدرس برگشت تابع (return address) یا حتی ساختار Stack خراب بشه.

این اتفاق می‌تونه باعث crash بشه یا حتی توسط مهاجم برای اجرای کد مخرب استفاده بشه.
